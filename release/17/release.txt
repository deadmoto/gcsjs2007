* новое поле в таблице hist "id_heating", отвечает за тип отопления (для фильтра, и для отчета по фактическим расходам). При вводе специалистом обязательно выбирать тип отопления правильный для соответствующего клиента. (тип отопления выбирается на форме редактирования клиента->Тарифы.
)
* процедура "updateHist_setwood.sql", проставит тип отопления для всех клиентов (центральное, твердое топливо). Если есть клиенты с газовым или электрическим отопление, то выставить тип отопления вручную.
* появилась возможность фильтровать записи по типу отопления. 
* новая процедура для сервера "getclfactsum" (для отчета).
* новый отчет "Округ->Фактические расходы", выбирает всех клиентов у кого первый месяц и тип аттестации "переаттестация" за выбранный месяц, последний активный период у клиента, сумму субсидии, сумму фактических расходов и разницу субсидии и ф. расходов. Для клиентов у кого тип отопления отличный от центрального период выбирается = 2 срока(1 год).
* скрипт "Heating.sql" создаст новую таблицу с идентичным названием, в которой хранится типы отопления.

-- если данные по фактическим расходам добавляются для клиента у кого не центральное отопление, то необходимо заполнить все имеющиеся периоды, если нет информации по фактическим расходам за прошлые периоды, то заполнить их нулями (для этого на форме редактирования фактических расходов просто выбрать прошлый период и нажать кнопку добавить). Без добавления прошлых периодов, отчет по фактическим расходам, для клиентов с нецентральным отоплением, может сойтись не верно (т.е. если у клиента 5 периодов, а добавить только 2 последних, исключая первые 3, то программа будет думать что 2 добавленных периода, относятся к 1 году).
